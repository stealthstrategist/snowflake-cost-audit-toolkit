## Common findings reference

**Warehouse too large**
- Finding: X-LARGE warehouse running queries that avg under 1GB scanned
- Fix: Downsize to SMALL or MEDIUM, monitor for 1 week
- Effort: Low

**Auto-suspend too high**
- Finding: Warehouse averaging 10+ minute sessions with low query frequency
- Fix: Set AUTO_SUSPEND = 60 on that warehouse
- Effort: Low

**No resource monitor**
- Finding: No spend cap on warehouse — one bad job can burn unlimited credits
- Fix: Run resource_monitor.sql and apply caps
- Effort: Low

**Time travel storage explosion**
- Finding: TIME_TRAVEL_BYTES >> ACTIVE_BYTES on high-churn tables
- Fix: Reduce DATA_RETENTION_TIME on those specific tables
- Effort: Medium

**Full table scans**
- Finding: High GB scanned per query on large tables with no clustering
- Fix: Add clustering key or search optimization on filter columns
- Effort: High
